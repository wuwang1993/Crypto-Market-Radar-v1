# 交易所币对监控 Telegram 推送系统开发方案

## 1. 项目定位

本项目是一个 **交易所币对行情监控 + 指标分析 + Telegram 自动推送系统**。

系统不做后台页面，不做网页管理，不做用户系统，不做自动交易。
核心目标是让程序在服务器上持续运行，实时监控指定币对行情，并在出现异常涨跌、买盘增强、卖盘增强、放量突破、放量下破、合约风险等情况时，自动推送到 Telegram。

---

## 2. 项目核心目标

```text
交易所行情数据
    ↓
实时采集
    ↓
指标计算
    ↓
行情状态判断
    ↓
告警规则触发
    ↓
Telegram 推送
```

系统需要实现：

1. 实时监控指定币对涨跌情况。
2. 判断短线是上涨、下跌、震荡还是放量突破。
3. 判断当前买盘强还是卖盘强。
4. 判断成交量是否异常放大。
5. 判断盘口买盘 / 卖盘是否异常。
6. 判断合约市场是否存在资金费率、持仓量、爆仓风险。
7. 按规则向 Telegram 推送提醒。
8. 定时向 Telegram 推送市场摘要。
9. 系统异常时自动推送运维提醒。
10. 为后续自动交易 Bot 提供数据基础。

## 6. 系统整体架构

```text
Crypto Market Radar Bot

交易所 WebSocket / REST API
        ↓
Exchange Adapter 交易所适配层
        ↓
Market Data 行情数据层
        ↓
Indicator Engine 指标计算层
        ↓
State Engine 状态判断层
        ↓
Alert Engine 告警规则层
        ↓
Telegram Push 推送层
        ↓
本地日志 / SQLite / 系统状态记录
```

---

## 7. 模块划分

系统拆分为 8 个核心模块：

| 模块            | 作用                       |
| ------------- | ------------------------ |
| 配置模块          | 读取交易所、币对、阈值、TG 配置        |
| 交易所数据模块       | 接入 WebSocket / REST，获取行情 |
| 数据缓存模块        | 保存最新行情、最近窗口数据            |
| 指标计算模块        | 计算涨跌幅、成交量、买卖量比、CVD 等     |
| 状态判断模块        | 判断强势上涨、放量下跌、买盘增强等        |
| 告警模块          | 判断是否触发提醒，并做去重限频          |
| Telegram 推送模块 | 发送 TG 消息                 |
| 日志与健康检查模块     | 记录运行状态、异常、断线、推送失败        |

# 9. 核心指标设计

## 9.1 涨跌幅指标

每个币对需要计算：

```text
1m 涨跌幅
3m 涨跌幅
5m 涨跌幅
15m 涨跌幅
1h 涨跌幅
4h 涨跌幅
24h 涨跌幅
```

判断规则：

| 条件           | 状态   |
| ------------ | ---- |
| 5m 涨幅 > 1%   | 快速上涨 |
| 5m 跌幅 < -1%  | 快速下跌 |
| 15m 涨幅 > 2%  | 短线强势 |
| 15m 跌幅 < -2% | 短线弱势 |
| 1h 涨幅 > 3%   | 日内强势 |
| 1h 跌幅 < -3%  | 日内弱势 |

---

## 9.2 成交量指标

需要计算：

```text
当前 1m 成交量
当前 5m 成交量
当前 15m 成交量
近 20 根 K 线平均成交量
成交量放大倍数
成交额变化
```

成交量放大倍数：

```text
当前成交量 / 过去 20 周期平均成交量
```

判断规则：

|      放大倍数 | 状态   |
| --------: | ---- |
|     < 0.8 | 缩量   |
| 0.8 - 1.2 | 正常   |
| 1.2 - 2.0 | 温和放量 |
| 2.0 - 4.0 | 明显放量 |
|     > 4.0 | 异常放量 |

---

## 9.3 主动买入 / 主动卖出指标

这是判断买卖状态的核心。

需要统计：

```text
主动买入成交量
主动卖出成交量
主动买入成交额
主动卖出成交额
买卖量比
净主动买入量
大单主动买入数量
大单主动卖出数量
```

买卖量比：

```text
主动买入量 / 主动卖出量
```

判断规则：

|      买卖量比 | 状态   |
| --------: | ---- |
|     > 2.0 | 买盘极强 |
| 1.5 - 2.0 | 买盘强  |
| 1.1 - 1.5 | 买盘略强 |
| 0.9 - 1.1 | 买卖均衡 |
| 0.6 - 0.9 | 卖盘略强 |
| 0.4 - 0.6 | 卖盘强  |
|     < 0.4 | 卖盘极强 |

---

## 9.4 CVD 指标

CVD 是累计成交量差，用于判断资金净流向。

```text
CVD = 累计主动买入量 - 累计主动卖出量
```

需要计算：

```text
1m CVD
5m CVD
15m CVD
1h CVD
CVD 变化方向
价格与 CVD 是否背离
```

判断规则：

| 价格 | CVD | 判断    |
| -- | --- | ----- |
| 上涨 | 上升  | 真买盘推动 |
| 上涨 | 下降  | 可能是假拉 |
| 下跌 | 下降  | 真卖盘推动 |
| 下跌 | 上升  | 可能有承接 |
| 横盘 | 上升  | 暗中买入  |
| 横盘 | 下降  | 暗中出货  |

---

## 9.5 盘口深度指标

需要计算：

```text
买一价
卖一价
Spread
0.1% 买盘深度
0.1% 卖盘深度
0.5% 买盘深度
0.5% 卖盘深度
1% 买盘深度
1% 卖盘深度
买卖深度比
最大买墙
最大卖墙
```

买卖深度比：

```text
买盘深度 / 卖盘深度
```

判断规则：

|     买卖深度比 | 状态     |
| --------: | ------ |
|     > 2.0 | 买盘挂单很强 |
| 1.3 - 2.0 | 买盘较强   |
| 0.8 - 1.3 | 盘口均衡   |
| 0.5 - 0.8 | 卖压较强   |
|     < 0.5 | 卖压很强   |

注意：

```text
盘口挂单可以撤单，不能单独作为买卖依据。
成交明细和 CVD 比盘口更真实。
```

---

## 9.6 VWAP 指标

VWAP 用于判断当前价格是否偏离平均成交成本。

用途：

```text
判断价格是否追高
判断回调支撑
判断跌破强弱线
判断趋势是否健康
```

判断规则：

| 状态             | 含义     |
| -------------- | ------ |
| 价格高于 VWAP      | 日内偏强   |
| 价格低于 VWAP      | 日内偏弱   |
| 价格远高于 VWAP     | 有追高风险  |
| 价格回踩 VWAP 反弹   | 买盘承接较好 |
| 价格跌破 VWAP 无法收回 | 趋势转弱   |

---

## 9.7 均线指标

建议计算：

```text
EMA7
EMA20
EMA60
MA20
MA60
```

判断规则：

| 条件                 | 状态   |
| ------------------ | ---- |
| 价格 > EMA20 > EMA60 | 上涨趋势 |
| 价格 < EMA20 < EMA60 | 下跌趋势 |
| EMA20 和 EMA60 缠绕   | 震荡   |
| 价格突破 EMA20         | 短线转强 |
| 价格跌破 EMA20         | 短线转弱 |

---

## 9.8 RSI 指标

建议计算：

```text
RSI 6
RSI 14
RSI 24
```

判断规则：

|     RSI | 状态   |
| ------: | ---- |
|    > 80 | 严重超买 |
| 70 - 80 | 超买   |
| 50 - 70 | 偏强   |
| 30 - 50 | 偏弱   |
| 20 - 30 | 超卖   |
|    < 20 | 严重超卖 |

RSI 只做风险辅助，不单独决定买卖。

# 10. 合约指标设计

## 10.1 资金费率 Funding Rate

需要展示：

```text
当前资金费率
下一次结算时间
最近 8 小时资金费率
最近 24 小时平均资金费率
资金费率变化趋势
```

判断规则：

|     资金费率 | 状态     |
| -------: | ------ |
|  > 0.05% | 多头较拥挤  |
|   > 0.1% | 多头严重拥挤 |
| < -0.05% | 空头较拥挤  |
|  < -0.1% | 空头严重拥挤 |

---

## 10.2 持仓量 OI

需要计算：

```text
当前 OI
5m OI 变化
15m OI 变化
1h OI 变化
OI 与价格关系
```

判断规则：

| 价格 | OI | 判断             |
| -- | -- | -------------- |
| 上涨 | 增加 | 新多进场，趋势较真      |
| 上涨 | 减少 | 空头平仓，持续性可能弱    |
| 下跌 | 增加 | 新空进场，偏空        |
| 下跌 | 减少 | 多头平仓 / 爆仓，风险释放 |
| 横盘 | 增加 | 多空加仓，可能快变盘     |
| 横盘 | 减少 | 市场降温           |

---

## 10.3 爆仓数据

需要计算：

```text
5m 多单爆仓金额
5m 空单爆仓金额
15m 多单爆仓金额
15m 空单爆仓金额
1h 爆仓金额
爆仓方向
爆仓强度
```

判断规则：

| 爆仓状态      | 含义     |
| --------- | ------ |
| 空单大量爆仓    | 上涨挤空   |
| 多单大量爆仓    | 下跌踩踏   |
| 爆仓后价格止跌   | 可能短线企稳 |
| 爆仓后继续下跌   | 趋势仍弱   |
| 上涨中空单连续爆仓 | 强势上涨   |
| 下跌中多单连续爆仓 | 强势下跌   |

# 11. 行情状态判断设计

每个币对最终需要输出一个状态。

## 11.1 状态类型

```text
强势上涨
弱势上涨
震荡偏多
震荡
震荡偏空
弱势下跌
强势下跌
放量突破
放量下破
高位滞涨
低位止跌
买盘增强
卖盘增强
异常波动
禁止交易
```

---

## 11.2 强势上涨判断

触发条件：

```text
5m 涨幅 > 0.5%
15m 涨幅 > 1%
成交量放大倍数 > 1.5
买卖量比 > 1.5
CVD 上升
价格 > VWAP
价格 > EMA20
```

输出状态：

```text
状态：强势上涨
方向：偏多
风险：追高风险中等
提示：已有仓位可持有，新仓等待回调
```

---

## 11.3 强势下跌判断

触发条件：

```text
5m 跌幅 < -0.5%
15m 跌幅 < -1%
成交量放大倍数 > 1.5
买卖量比 < 0.6
CVD 下降
价格 < VWAP
价格 < EMA20
```

输出状态：

```text
状态：强势下跌
方向：偏空
风险：不要接刀
提示：等待卖压衰减
```

---

## 11.4 放量突破判断

触发条件：

```text
价格突破最近 20 根 K 线高点
成交量放大倍数 > 2
买卖量比 > 1.5
CVD 同步上升
卖盘深度被吃掉
```

输出状态：

```text
状态：放量突破
方向：短线偏多
风险：追高风险存在
提示：等待回踩确认
```

---

## 11.5 放量下破判断

触发条件：

```text
价格跌破最近 20 根 K 线低点
成交量放大倍数 > 2
买卖量比 < 0.6
CVD 同步下降
买盘深度变薄
```

输出状态：

```text
状态：放量下破
方向：短线偏空
风险：可能继续加速下跌
提示：禁止盲目抄底
```

---

## 11.6 高位滞涨判断

触发条件：

```text
价格处于 1h 高位
RSI > 70
价格继续上涨但 CVD 不再上涨
主动买入减少
主动卖出增加
上方卖盘深度增加
```

输出状态：

```text
状态：高位滞涨
方向：多头动能减弱
提示：已有仓位分批止盈，新仓不追
```

---

## 11.7 低位止跌判断

触发条件：

```text
价格处于 1h 低位
RSI < 30
下跌速度放缓
CVD 不再下降
主动卖出减少
买盘深度增加
价格重新站上短周期均线
```

输出状态：

```text
状态：低位止跌
方向：可能反弹
提示：观察反弹确认，小仓试探
```

# 12. 告警规则设计

## 12.1 告警等级

| 等级 | 名称   | 说明         |
| -- | ---- | ---------- |
| L1 | 普通提醒 | 定时摘要、轻微信号  |
| L2 | 重要提醒 | 快速涨跌、买卖盘增强 |
| L3 | 风险提醒 | 合约拥挤、盘口异常  |
| L4 | 紧急提醒 | 极端波动、系统异常  |

---

## 12.2 快速上涨提醒

触发条件：

```text
5m 涨幅 > 1%
成交量放大 > 1.5 倍
买卖量比 > 1.3
```

推送模板：

```text
🚀 快速上涨提醒

币对：BTC/USDT
价格：104,200
5m涨幅：+1.24%
15m涨幅：+2.08%
成交量：放大 2.1 倍
买卖量比：1.82
CVD：持续上升
盘口：买盘较强

系统判断：
短线买盘明显增强，属于强势上涨。

操作提示：
不建议直接追高，等待回调 VWAP / EMA20 后观察。
```

---

## 12.3 快速下跌提醒

触发条件：

```text
5m 跌幅 < -1%
成交量放大 > 1.5 倍
买卖量比 < 0.7
```

推送模板：

```text
🔻 快速下跌提醒

币对：SOL/USDT
价格：158.2
5m跌幅：-1.46%
15m跌幅：-2.32%
成交量：放大 2.8 倍
买卖量比：0.48
CVD：快速下降
盘口：买盘变薄

系统判断：
主动卖出明显增强，短线卖压较重。

操作提示：
不要盲目接刀，等待卖压减弱。
```

---

## 12.4 买盘增强提醒

触发条件：

```text
买卖量比 > 1.8
CVD 连续上升
价格重新站上 VWAP
```

推送模板：

## 14.1 冷却规则

| 告警类型   | 冷却时间 |
| ------ | ---: |
| 快速上涨   |  5分钟 |
| 快速下跌   |  5分钟 |
| 买盘增强   | 10分钟 |
| 卖盘增强   | 10分钟 |
| 放量突破   | 15分钟 |
| 放量下破   | 15分钟 |
| CVD 背离 | 10分钟 |

| 系统异常   |  5分钟 |

---

## 14.2 去重规则

同一个币对、同一种告警，在冷却时间内只推一次。

唯一键：

```text
币对 + 告警类型 + 时间窗口
```

示例：

```text
BTC/USDT + 快速上涨 + 5m
```

---

## 14.3 合并推送

如果多个币同时触发普通提醒，合并为一条。

```text

本地存储是否正常

程序是否卡死
```

| 异常           | 处理方式         |
| ------------ | ------------ |

| TG 推送失败      | 重试并记录        |
| 程序崩溃         | systemd 自动重启 |

# 26. 风险与注意事项

## 26.1 数据风险

```text
交易所接口可能断线
WebSocket 数据可能延迟
REST API 可能限流
盘口数据可能被假挂单干扰
成交明细比盘口更可靠
```

## 26.2 策略风险

```text
系统提醒不等于一定可以买卖
买盘增强不代表一定上涨
卖盘增强不代表一定继续下跌
RSI 超买不代表马上跌
RSI 超卖不代表马上涨
合约高杠杆风险极高
```

## 26.3 推送风险

```text
提醒太频繁会刷屏
提醒太少会错过行情
阈值需要根据币种波动调整
山寨币阈值应高于 BTC/ETH
```
